# 通用编码约束库（与语言无关）

> 本文件列出**与编程语言无关**的通用编码约束。Phase 1 生成规格书 §9 时，按方案实际涉及范围**摘取**条款落地，每条都必须可机械验证。
>
> **基本原则**：日志/HTTP/中间件/测试库等**风格类**约束**不写在本文件**，统一从 `.qiqskills/spec2code/REPO_PROFILE.md` 的"风格基线"小节中沿用仓库现状，避免与现仓库冲突。本文件只保留"违反必返工"的最小核心集，其它约束按需后续扩展。

## A. 类型与单位

- **A1** 金额字段统一使用整型，单位为"分"，禁止使用浮点数。
- **A2** 时间字段统一使用 UTC，DB 列类型精确到毫秒；展示层负责时区转换。
- **A3** ID 字段统一使用 64 位整型；外部展示如需字符串（避免 JS 精度丢失）必须显式转字符串。
- **A4** 枚举值统一以"显式常量 + 显式数值"定义，禁止使用魔法数字。

## B. 超时与外部调用

- **B1** 所有数据库操作必须设置超时（默认 3 秒，可被规格书覆盖）。
- **B2** 所有 RPC/HTTP 调用必须设置超时（默认 1 秒，可被规格书覆盖）。
- **B3** 所有外部 I/O 必须传递 `context`（或等价 cancellation 机制）。
- **B4** 仅幂等的下游调用允许重试；非幂等下游禁止重试，必须显式标注。

## C. 幂等与并发

- **C1** 所有写操作必须幂等。
- **C2** 幂等键的生成、存储、TTL 在规格书中显式规定。
- **C3** 乐观锁使用版本号字段，更新时 `WHERE version=? SET version=version+1`，并校验影响行数。
- **C4** 禁止在 for 循环内做数据库/RPC 调用，必须使用批量接口或显式声明 N+1 风险已评估。

## D. 错误处理

- **D1** 错误处理必须保留完整错误链（语言对应的 wrap 机制）。
- **D2** 禁止吞掉错误（仅在显式标注"可忽略"的场景下允许，并必须打日志）。
- **D3** 每种业务错误对应一个明确的错误码 + 调用方处理建议。
- **D4** 系统级错误（DB 连接断开、超时）与业务错误（如库存不足）必须区分类型。
